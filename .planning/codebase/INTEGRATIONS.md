# External Integrations

**Analysis Date:** 2026-04-15

## APIs & External Services

**Ollama (Local LLM & Embeddings):**
- LLM Chat Completions - For RAG-powered chat responses
  - SDK/Client: `ollama` Python package (>=0.1.0)
  - Default model: `llama3:8b` (configurable via `/api/model/select` endpoint)
  - API Endpoint: Expected at `http://localhost:11434` (Ollama default)
  - Used by: `backend/ollama_client.py`, `backend/services/rag_service.py`, `backend/api/chat.py`

- Text Embeddings - For document semantic search
  - Model: `nomic-embed-text`
  - Expected dimensions: 768 or 1024
  - API Endpoint: Same Ollama service
  - Used by: `backend/services/embedding_service.py`
  - Integration point: `backend/api/documents.py` → `embedding_service.generate_embeddings()`

**Ollama Status & Health:**
- Endpoint: `GET /api/ollama/status`
- Checks Ollama connectivity and available models
- Includes test completion with selected model
- Response: Status, models list, test result, and model recommendation

## Data Storage

**Databases:**

**SQLite (Chat Sessions):**
- Location: `backend/sessions.db` (auto-created on first run)
- ORM: SQLAlchemy 2.0+
- Connection: `sqlite:///./sessions.db` with timeout=20.0
- Purpose: Persistent chat session storage
- Table: `chat_sessions` with schema defined in `backend/models/session.py`
- Fields:
  - `id` (String, PK): UUID session identifier
  - `created_at` (String): ISO format timestamp
  - `updated_at` (String): ISO format timestamp
  - `messages` (JSON): Array of message objects with role and content
- Access: Via `backend/services/session_service.py` for CRUD operations
- Dependency injection: `get_db()` function in `backend/models/session.py`

**Vector Storage (ChromaDB):**
- Type: Persistent vector database
- Location: `backend/uploads/vectors/` (auto-created)
- Client: `chromadb` >=0.4.0 PersistentClient
- Collection: `documents` with cosine similarity metric
- Purpose: Store document chunk embeddings for semantic search
- Configuration:
  - Anonymized telemetry disabled
  - HNSW space set to cosine
- Used by:
  - `backend/services/vector_service.py` for storage and retrieval
  - `backend/api/documents.py` for indexing after upload
  - `backend/services/retrieval_service.py` for search queries
- Schema: Chunks stored with doc_id, filename, and 1024-dim embeddings (nomic-embed-text)

**File Storage:**

**Local Filesystem:**
- Location: `backend/uploads/` (auto-created)
- Purpose: Store uploaded documents (PDF, TXT, MD files)
- Max file size: 50 MB (enforced in `backend/services/file_validator.py`)
- File naming: UUID-based unique filenames to prevent collisions
- Example: `{uuid}.pdf`, `{uuid}.txt`, `{uuid}.md`
- Subdirectories:
  - `backend/uploads/vectors/` - ChromaDB vector data

**Caching:**
- None detected. Session and vector data are persistent storage only.

## Authentication & Identity

**Auth Provider:**
- Custom/Frontend-managed sessions (no external auth service)
- Session ID: UUID generated by frontend
- Session creation: Frontend generates UUID and posts to `POST /chat/session/new` or `GET /chat/session/{session_id}`
- Session persistence: Backend stores in SQLite with full message history
- No API key or token requirement for chat endpoints
