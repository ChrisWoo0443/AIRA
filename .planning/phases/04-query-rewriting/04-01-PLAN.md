    **Step 3: Implement query_rewrite_service.py (TDD green).**
    Create `backend/services/query_rewrite_service.py` following the contextual_retrieval_service.py pattern.

    Module structure:
    1. Imports: `ollama`, `logging`, `math`, `pydantic.BaseModel`, `enum.Enum`, `dataclasses.dataclass`, `typing.Optional`
    2. Import from config: all `QUERY_REWRITE_*`, `CONFIDENCE_GATE_*`, `HYDE_*` constants
    3. Import from services: `generate_embeddings` from `embedding_service`, `get_selected_model` from `ollama_client`
    4. Logger: `logger = logging.getLogger(__name__)`

    Data models (per D-01):
    - `QueryType(str, Enum)`: standalone, follow_up, abstract
    - `QueryClassification(BaseModel)`: query_type: QueryType, reasoning: str
    - `RewriteResult` dataclass: original_query, effective_query, query_type, rewritten_query (Optional), hyde_passage (Optional), hyde_embedding (Optional[list[float]]), confidence_score (Optional[float]), used_fallback (bool)

    Prompt templates (from RESEARCH.md):
    - `CLASSIFICATION_PROMPT`: classifies as standalone/follow_up/abstract with explicit pivot handling
    - `REWRITE_PROMPT`: resolves references minimally, does not add information
    - `HYDE_PROMPT`: generates 3-5 sentence hypothetical passage

    Functions:
    - `_get_rewrite_model() -> str`: returns QUERY_REWRITE_MODEL or get_selected_model()
    - `_format_history(conversation_history: list[dict], window: int) -> str`: takes last `window` messages, formats as "User: ...\nAssistant: ..." text
    - `_cosine_similarity(vec_a: list[float], vec_b: list[float]) -> float`: dot product / (mag_a * mag_b), returns 0.0 on zero magnitude
    - `classify_query(query: str, conversation_history: list[dict]) -> QueryClassification`: short-circuits if history empty; calls ollama.chat with format=QueryClassification.model_json_schema(), temperature=0, num_predict=100; parses with model_validate_json; falls back to standalone on any exception
    - `rewrite_followup(query: str, conversation_history: list[dict]) -> str`: calls ollama.chat with REWRITE_PROMPT formatted with history and query; returns response content stripped; falls back to original query on exception
    - `generate_hyde_passage(query: str) -> Optional[str]`: calls ollama.chat with HYDE_PROMPT, num_predict=HYDE_PASSAGE_MAX_TOKENS; returns passage text; returns None on exception
    - `confidence_gate(original_query: str, rewritten_query: str, threshold: float) -> tuple[str, Optional[float]]`: calls generate_embeddings for both queries, computes cosine similarity, returns (rewritten_query, similarity) if >= threshold, else (original_query, similarity) with logger.warning; returns (original_query, None) on embedding failure
    - `rewrite_query(query: str, conversation_history: list[dict]) -> RewriteResult`: the main orchestrator:
      1. Classify the query
      2. If standalone: return RewriteResult with effective_query=original
      3. If follow_up: rewrite, confidence gate (CONFIDENCE_GATE_THRESHOLD), build result
      4. If abstract: generate HyDE passage, if passage exists embed it via generate_embeddings, confidence gate the passage against original (HYDE_CONFIDENCE_GATE_THRESHOLD), build result with hyde_embedding
      5. Wrap entire function in try/except: on any unhandled exception, log warning, return RewriteResult with effective_query=original and used_fallback=True

    Key implementation details:
    - Use `ollama.chat(model=model, messages=[...], format=QueryClassification.model_json_schema(), options={"temperature": 0, "num_predict": 100})` for classification (per D-01 locked decision)
    - Use `ollama.chat(model=model, messages=[...], options={"temperature": 0})` for rewriting (deterministic)
    - Use `ollama.chat(model=model, messages=[...], options={"temperature": 0.7, "num_predict": HYDE_PASSAGE_MAX_TOKENS})` for HyDE (slight creativity)
    - History window: slice `conversation_history[-QUERY_REWRITE_HISTORY_WINDOW:]` (per D-02)
    - All Ollama calls wrapped in try/except with graceful fallback (per established pattern)
