    **Step 2: Modify retrieval_service.py search_documents signature.**
    Add `query_embedding: Optional[list[float]] = None` parameter. When provided, skip the `generate_embeddings([query])` call and use the provided embedding directly. This is the HyDE path -- the embedding was already computed from the hypothetical passage.

    Current code (lines 226-255):
    ```python
    def search_documents(query, top_k=5, doc_ids=None):
        try:
            query_embeddings = generate_embeddings([query])
            query_embedding = query_embeddings[0]
        except RuntimeError as e:
            raise RuntimeError(f"Cannot search: {str(e)}") from e
    ```

    Change to:
    ```python
    def search_documents(query, top_k=5, doc_ids=None, query_embedding=None):
        if query_embedding is None:
            try:
                query_embeddings = generate_embeddings([query])
                query_embedding = query_embeddings[0]
            except RuntimeError as e:
                raise RuntimeError(f"Cannot search: {str(e)}") from e
    ```

    The rest of the function uses `query_embedding` unchanged. No other modifications needed.

    **Step 3: Modify rag_service.py to call query rewriting.**
    Add imports at the top of the file:
    ```python
    from config import QUERY_REWRITING_ENABLED
    from services.query_rewrite_service import rewrite_query
    ```

    Add a `logger = logging.getLogger(__name__)` (import logging if not already imported).

    Replace line 79 (`search_results = search_documents(query, top_k=top_k, doc_ids=document_ids)`) with:
    ```python
    # Query rewriting: resolve follow-ups and abstract queries before search
    effective_query = query
    hyde_embedding = None

    if QUERY_REWRITING_ENABLED and conversation_history:
        try:
            rewrite_result = rewrite_query(query, conversation_history)
            effective_query = rewrite_result.effective_query
            hyde_embedding = rewrite_result.hyde_embedding
        except Exception as exc:
            logger.warning("Query rewriting failed: %s, using original query", str(exc))

    # Retrieve relevant document chunks
    search_results = search_documents(
        effective_query,
        top_k=top_k,
        doc_ids=document_ids,
        query_embedding=hyde_embedding,
    )
    ```

    Note the optimization: skip rewrite_query entirely when `conversation_history` is empty (same short-circuit as classify_query, but avoids the function call overhead). This ensures first-message-in-session queries have zero added latency per Pitfall 3.

    Keep the existing `query` variable for the user message in the LLM prompt (line 124: `messages.append({"role": "user", "content": query})`). The rewritten query is only for search, not for the LLM conversation.
